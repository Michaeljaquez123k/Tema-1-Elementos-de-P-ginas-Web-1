La World Wide Web (WWW) fue creada por Tim Berners-Lee en 1989 mientras trabajaba en el CERN, el laboratorio de física de partículas en Suiza. Berners-Lee desarrolló el lenguaje HTML (HyperText Markup Language) y el protocolo HTTP (Hypertext Transfer Protocol) para permitir la creación y el intercambio de información en forma de páginas web.

La Web 2.0 es una evolución de la web que se caracteriza por la participación activa de los usuarios y la creación de contenidos colaborativos. En la Web 2.0, los usuarios pueden interactuar entre sí, compartir información, crear y editar contenido en línea, participar en redes sociales y utilizar aplicaciones web. Algunos ejemplos de servicios y plataformas de la Web 2.0 incluyen las redes sociales como Facebook y Twitter, los blogs, los wikis y los sitios de compartición de vídeos como YouTube.

La Web 3.0, también conocida como la web semántica, es una evolución de la web que busca dotar de mayor significado y contexto a la información en línea. La Web 3.0 utiliza tecnologías como el lenguaje de marcado semántico (RDF), ontologías y algoritmos de inteligencia artificial para comprender y organizar la información de manera más precisa. Esto permite que las máquinas puedan entender el contenido de las páginas web y proporcionar resultados de búsqueda más relevantes y personalizados. La Web 3.0 también se enfoca en la interoperabilidad de datos entre diferentes aplicaciones y servicios en línea

El W3C (World Wide Web Consortium) es una organización internacional que desarrolla estándares y guías para la web. Fue fundada por Tim Berners-Lee en 1994 y tiene como objetivo promover la interoperabilidad y la evolución de la web. El W3C establece estándares abiertos y recomendaciones técnicas para garantizar que los sitios web sean accesibles, interoperables y compatibles con diferentes navegadores y dispositivos. Algunos de los estándares desarrollados por el W3C incluyen HTML, CSS (Hojas de estilo en cascada) y el estándar de accesibilidad WCAG (Pautas de accesibilidad al contenido web).

Lenguaje de etiquetas

SGML (Standard Generalized Markup Language) es un lenguaje de marcado generalizado que se utiliza para la organización y etiquetado de documentos. Fue desarrollado en la década de 1980 y se utiliza como base para otros lenguajes de etiquetas, como HTML y XML. SGML permite definir la estructura y el formato de un documento mediante etiquetas y atributos, lo que facilita la creación y el intercambio de información estructurada.

HTML (Lenguaje de marcado de hipertexto) es el lenguaje de marcado utilizado para crear páginas web. HTML utiliza etiquetas para definir la estructura y el contenido de una página web, como encabezados, párrafos, enlaces, imágenes y tablas. Las etiquetas de HTML se utilizan para dar formato al texto, insertar elementos multimedia y crear enlaces entre diferentes páginas web. III. Otros lenguajes relacionados al desarrollo

CSS/CSS3 : CSS (Hojas de estilo en cascada) es un lenguaje utilizado para definir el estilo y la apariencia de una página web. Permite controlar el diseño, el formato y la presentación de los elementos HTML. CSS3 es la versión más reciente de CSS y ofrece nuevas características y funcionalidades para el diseño web.

JavaScript : JavaScript es un lenguaje de programación utilizado para agregar interactividad y funcionalidad a las páginas web. Permite manipular el contenido de la página, responder a eventos del usuario, realizar validaciones de formularios, crear animaciones y mucho más. Es ampliamente utilizado en el desarrollo web tanto en el lado del cliente (navegador) como en el lado del servidor (Node.js)

Ajax : Ajax (JavaScript asíncrono y XML) es una técnica de desarrollo web que permite actualizar partes específicas de una página web sin tener que recargarla por completo. Utiliza JavaScript para enviar y recibir datos en un segundo plano, lo que mejora la experiencia del usuario y la velocidad de carga de la página.

Tecnologías y lenguajes del lado del servidor : En el desarrollo web, existen varios lenguajes y tecnologías utilizadas en el lado del servidor para procesar la lógica de negocio, interactuar con bases de datos y generar contenido dinámico. Algunos ejemplos de lenguajes del lado del servidor son PHP, Python, ASP.NET y Ruby. Estos lenguajes permiten crear aplicaciones web más complejas y dinámicas.

Navegadores : Los navegadores web son aplicaciones que permiten a los usuarios acceder y visualizar páginas web. Algunos de los navegadores más populares son Google Chrome, Mozilla Firefox, Safari, Microsoft Edge y Opera. Cada navegador puede tener su propia interpretación y soporte de los estándares web, por lo que es importante asegurarse de que las páginas web sean compatibles con diferentes navegadores.

Motores renderizados : Los motores de renderizado son componentes de los navegadores web que se encargan de interpretar y mostrar el contenido de las páginas web. Algunos ejemplos de motores de renderizado son Blink (utilizado por Google Chrome), Gecko (utilizado por Mozilla Firefox), WebKit (utilizado por Safari) y Trident (utilizado por versiones antiguas de Internet Explorer). Cada motor de renderizado tiene su propia forma de interpretar y mostrar el contenido web, lo que puede afectar la apariencia y el rendimiento de las páginas.

IV. Herramientas de desarrollo 1. Software

Software WYSIWYG : El software WYSIWYG (What You See Is What You Get) es una herramienta que permite a los desarrolladores crear y diseñar páginas web visualmente, mostrando el resultado final mientras se trabaja en ella. Estas herramientas facilitan la creación de sitios web sin necesidad de escribir código manualmente, ya que permiten arrastrar y soltar elementos y aplicar estilos de forma visual.

Otras alternativas :Figma : Figma es una herramienta de diseño colaborativa basada en la nube que permite a los equipos de desarrollo crear y prototipar interfaces de usuario de forma eficiente. Permite la colaboración en tiempo real y ofrece características como diseño de componentes reutilizables y animaciones interactivas.

InVision : InVision es una plataforma de diseño y prototipado que permite a los equipos de desarrollo crear y compartir prototipos interactivos de alta fidelidad. Facilita la colaboración entre diseñadores y desarrolladores, y ofrece herramientas para recopilar comentarios y realizar pruebas de usabilidad

Zeplin : Zeplin es una herramienta de colaboración para diseñadores y desarrolladores que facilita la comunicación y la transferencia de diseños desde herramientas de diseño como Sketch o Adobe XD en el proceso de desarrollo. Inspección de código desde el navegador : Los navegadores web modernos ofrecen herramientas de desarrollo integradas que permiten a los desarrolladores inspeccionar y depurar el código de una página web en tiempo real. Estas herramientas proporcionan información detallada sobre la estructura del DOM, estilos aplicados, rendimiento de la página y permiten realizar pruebas y depuración de JavaScript.

Tema 2: El Diseño De Una Página Web

Páginas web corporativas : Son utilizadas por empresas y organizaciones para presentar información sobre sus productos, servicios, historia y contacto. Estas páginas suelen tener un diseño profesional y transmitir la identidad de la marca.

Tiendas en línea : También conocidas como sitios de comercio electrónico, estas páginas web permiten a los usuarios comprar productos o servicios en línea. Suelen incluir catálogos de productos, carritos de compra y opciones de pago seguro.

Blogs : Los blogs son páginas web donde se publican regularmente artículos o entradas sobre temas específicos. Suelen tener una estructura cronológica inversa, mostrando las entradas más recientes primero. Los lectores pueden dejar comentarios y compartir el contenido en redes sociales.

II. Puntos básicos en la elaboración de una página Web.

Idea y planificación: Es importante tener una idea clara de lo que se quiere lograr con la página web y planificar su estructura y contenido de manera adecuada.

Diseño: El diseño de la página web debe ser atractivo, fácil de navegar y coherente con la identidad de la marca o el propósito de la página.

Elección de la plataforma: Existen diferentes plataformas y herramientas disponibles para crear una página web. Es importante elegir la que mejor se adapte a las necesidades y habilidades del desarrollador.

Tema 3: Estructura De Una Página Web

I. Jerarquía de directorio de un proyecto web Directorio raíz: Es el directorio principal del proyecto web y contiene todos los archivos y carpetas relacionados con el sitio.

Carpetas principales: Se pueden crear carpetas principales para organizar diferentes tipos de archivos, como "css" para los archivos de estilos CSS, "js" para los archivos de JavaScript, "img" para las imágenes.

II. Elementos de estructura para el diseño de una página web: Concepto

Regla utilizada para validar y certificar una página web: Una regla comúnmente utilizada para validar y certificar una página web es el estándar del Consorcio World Wide Web (W3C). El W3C establece pautas y especificaciones para el desarrollo web, incluyendo la validación del código HTML y CSS. La validación de una página web asegura que cumple con los estándares y recomendaciones establecidos, lo que ayuda a garantizar la compatibilidad y accesibilidad del sitio en diferentes navegadores y dispositivos.
